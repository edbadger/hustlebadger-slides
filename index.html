The problem

Decks eat time

Most of the work in a presentation is not thinking. It is dragging text boxes, fixing alignment, and re-pasting the same chart.

What you fight

Formatting that breaks the moment you edit. No real version history, just Deck_v7_FINAL_2.pptx. A slide you can't easily reuse anywhere else.

And the AI part

A .pptx is a zip of XML. AI can read it, but it can't really design in it. So you end up back nudging boxes by hand.

The reframe

A slide is just a web page

One section is one slide. A browser already knows how to lay out text, images, and columns. And Claude is very good at writing web pages.

What changes

You brief in plain words. Claude builds the HTML. You edit by asking, not by clicking.

What you get

A single HTML file, not a binary blob. The slide you are reading now was built exactly this way.

How it is built

The anatomy of an HTML deck

You don't need a framework. The whole thing is four ideas.

One section
per slide

Each slide is a <section>. Add one, you add a slide.

A fixed stage

A 16:9 box, 1280×720, centred and scaled to any screen.

Keyboard nav

Arrows, space, and click move between slides. About 20 lines of JS.

Tokens at
the top

Colours, fonts, and spacing as CSS variables in one place.

The method

Don't ask for twenty slides

Ask for the whole deck in one go and you get generic filler. Build it in three moves instead.

01
Outline first. Agree the structure: titles and one line per slide. Edit it before any design exists.
02
Build one slide. Get a single template slide looking right. That sets the vocabulary for the rest.
03
Generate in batches. Add the remaining slides against that template, a few at a time. Review, adjust, repeat.
Getting the look

Direct the design

Vague briefs average out into AI slop. Specific direction is what gets you a deck that looks deliberate.

Lock the canvas

State the aspect ratio and a fixed viewport: 16:9, 1280×720. Stops overflow and wandering sizes.

Hand it your brand

Give hex codes and font names as tokens. Name the fonts, or it defaults to generic.

Show, don't describe

Paste a screenshot of a layout you like and say "match this." It reads design faster than adjectives.

Brief by principle

"Editorial, generous whitespace, one accent colour" beats "make it look nice."

Reuse one template

Pass the slide you liked back in every time. Consistency comes from repetition.

Ask for print styles

Request @media print so the PDF export stays clean.

What goes wrong

Four traps, four fixes

Text overflows

Copy spills off the stage. Fix: less per slide; if it won't fit, split it.

Sizes drift

Every slide looks slightly different. Fix: lock one template and reuse it.

Fonts vanish offline

Web fonts fail with no internet. Fix: embed them, or fall back to a system stack.

Generic look

It feels like every other AI deck. Fix: brand tokens plus a reference screenshot.
